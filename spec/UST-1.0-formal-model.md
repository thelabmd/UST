## F.1 The measurable space of world-states

## F.2 Measurement vs reality вАФ the origin of "fixation, not truth"

Distinguish two maps into `V`:

- `YбµҐ,вВЬ : ќ© вЖТ V` вАФ the **true** coordinate (what the field actually is).
- `MбµҐ,вВЬ : ќ© вЖТ V` вАФ the **measurement** the publisher recorded (a sensor reading, a computation, a copy-paste).

In general `MбµҐ,вВЬ = m(YбµҐ,вВЬ, ќµбµҐ,вВЬ)` for some measurement map `m` and disturbance `ќµ` (a wrong sensor, a rounding,
a lie). `V = ќ£*` is a space of strings вАФ there is no addition on it; `ќµ` is an ABSTRACT deviation (the reading
differs from the truth in whatever way), not an additive noise term.

**A UST record binds `MбµҐ,вВЬ`, never `YбµҐ,вВЬ`.** It fixes *the measurement the publisher committed to, including its
error*. This is the entire content of "fixation, not truth" (¬І1): the protocol certifies `M`, and is silent on
the deviation between `M` and `Y`. Accountability follows вАФ you learn *which publisher committed which `M` at which `t`* вАФ but truth
(`Y`) is out of scope by construction.

## F.3 The anchor journal is a filtration

Let the external append-only log (the anchor substrate, ¬І11) define, for each real time `t`:

  **FвВЬ := ѕГ( Lбµ§ : u вЙ§ t ),**  the ѕГ-algebra generated by the JOURNAL COORDINATE SUB-PROCESS up to `t` вАФ the
  family of maps `{ Lбµ§ }_{u вЙ§ t}` from F.1, NOT the current set `LвВЬ` alone (a snapshot forgets WHEN each hash
  entered, and `ѕГ` of a single set does not recover the past вАФ the two-independent-red-team fix, 17th round).
  The atomic events are `E_{h,u} := { ѕЙ : h вИИ Lбµ§(ѕЙ) }`; "the log" is the explicit F.1 coordinate, no

**Proposition F.3 (FвВЬ is a filtration).** `{FвВЬ}` is a filtration: each `FвВЬ` is a ѕГ-algebra, and `s вЙ§ t вЗТ FвВЫ вКЖ FвВЬ`.

*Proof.* Each `FвВЬ` is generated by a family of measurable maps, hence a ѕГ-algebra. For `s вЙ§ t` the generating
FAMILY nests literally вАФ `{ Lбµ§ }_{u вЙ§ s} вКЖ { Lбµ§ }_{u вЙ§ t}` вАФ and the ѕГ-algebra generated by a larger family of
maps contains the one generated by a sub-family; therefore `FвВЫ вКЖ FвВЬ`. (This is the correct move: the earlier
"`LвВЫ вКЖ LвВЬ` hence `ѕГ(LвВЫ) вКЖ ѕГ(LвВЬ)`" was the classic `A вКЖ B вЗП ѕГ(A) вКЖ ѕГ(B)` trap вАФ repaired by generating from the
history `{Lбµ§}` rather than from one set.) вИО

**Corollary F.3.1 (un-backdatable time).** Backdating is the false claim that a hash `h` was present earlier
than it was. It is refuted not by a meta-statement about algebra membership but by an EVENT in the realized
history `ѕЙ*`: a historical checkpoint witnesses `ѕЙ* вИЙ E_{h,tвВА}` (h absent by `tвВА`) yet `ѕЙ* вИИ E_{h,tвВБ}` (present
by `tвВБ`), so the earliest provable existence is `tвВБ`, never `tвВА`. The index `t` of `{FвВЬ}` is REAL time as
attested by the anchor substrate (¬І11) вАФ the filtration is not merely ordered, it is TIME-STAMPED by an external
base the verifier
already trusts. Monotonicity alone orders events; the substrate's time base pins the order to the calendar вАФ
both together give un-backdatability, stated purely in EVENT form: the truth-values `ѕЙ* вИЙ E_{h,tвВА}` and
`ѕЙ* вИИ E_{h,tвВБ}` in the realized history (never "membership of an algebra", which is a meta-fact, not a world
event). One honest caveat (18th round): an inclusion-only Merkle log proves "existed BY `tвВБ`" but cannot by
itself prove "ABSENT at `tвВА`" вАФ refuting a backdating claim needs **authenticated non-membership** (a complete
verifiable snapshot at `tвВА`, or a non-membership proof). Without a valid `tвВА` receipt, backdating is
UNCONFIRMED; with authenticated non-membership it is REFUTED. (The reference journal is a per-hour COMPLETE
snapshot вАФ every slot of the hour listed вАФ so non-membership there is checkable by construction.) This is
exactly why a **retro-seal after an outage is fatal** вАФ against a snapshot-complete journal it is a provably
false claim about the realized history, not a mere policy breach.

## F.4 Streams are adapted processes

Model the publisher's emission as a **DocSpace-valued process** `RвВЬ : ќ© вЖТ DocSpace вИ™ {вК•}` (вК• = no frame at
`t`). Define the **commitment process** `HвВЬ(ѕЙ) := LвВЬ(ѕЙ)(frameвВЬ)` вАФ the journal's entry AT this frame (`HвВЬ := вК•`

cannot single out WHICH element is this frame's hash; the frame-indexed journal is what makes the commitment
process genuinely adapted вАФ 18th-round fix.) For a verifier the useful statement is the fixed-`d` form: for a
concrete received document `d`, the ANCHORING FACT `A_{d,t} := { ѕЙ : contentHash(d) вИИ range LвВЬ(ѕЙ) }` is an

this filtration. **Completeness over `[tвВА, tвВБ]`** is a RANGE property (a
verdict of `verifyStream` over an interval, ¬І11.3) вАФ a single document's tier never asserts it. It is the statement that
the observed adapted process has *no missing frame* in the interval вАФ an `F_{tвВБ}`-measurable assertion witnessed by
a covering checkpoint (`M5`). "The stream is complete" = "this adapted process is fully observed up to the
stopping time `tвВБ`."

**Completeness is authenticated coverage of a committed grid вАФ no-deletion is not no-omission (M5, made precise).**
The `prev`-chain establishes a TOTAL ORDER on the observed frames and forecloses DELETION from a shown chain
(removing frame `t` orphans `t+1`'s `prev`), but it does NOT by itself foreclose OMISSION: a publisher that never
emits frame `t` and links `t+1.prev = t-1` yields a self-consistent chain WITH A HOLE. Chain-consistency and

publisher's **cadence** `c_n(t)`, modelled as a signed, time-resolved parameter вАФ itself an adapted process,
resolved at `t` EXACTLY as the key-log resolves the active key at `t` (¬І12.2), so it cannot be shrunk post-hoc to
a coarser grid that hides slots. Then the expected grid over a closed interval is DETERMINISTIC,
`G(n,tвВА,tвВБ) := { the ust_id grid points of spacing c_n over [tвВА,tвВБ] }`, and `expected_slot_count := |G|` is a

without the signed cadence `|G|` is unknown, so `frame_count = |observed|` cannot be compared to `|G|`, and the
honest verdict is the strictly coarser **chain-consistent** (no-deletion), never **complete**. Note what does NOT
change: the checkpoint is the existing `class:"attestation"` with two interval bounds added to its value, the
grid is COMPUTED not stored, and the gap record already exists (¬І11.1) вАФ completeness is earned by adding the
cadence coordinate to `вДР`, not by any new document shape.

## F.5b Downgrade resistance is the consumer's floor, not the producer's promise

The tiers are totally ordered, `LIGHT < HIGH < TOP`, and by Theorem F.5 each is a coarser ѕГ-algebra than the
next: `рЭТЃ_LIGHT вКЖ рЭТЃ_HIGH вКЖ рЭТЃ_TOP`. Write `T(d, вДР)` for the FINEST tier decidable for document `d` from an
information set `вДР` вАФ the verdict the reference verifier emits. Three facts pin its behaviour under an attacker
who can only DELETE evidence (strip the anchor `proof`, omit the genesis / key-log, hide the name-map):

- **Monotone erosion.** `вДР' вКЖ вДР вЗТ T(d, вДР') вЙ§ T(d, вДР)`. Every tier above LIGHT is earned by a coordinate that
  lives OUTSIDE the document bytes вАФ the resolved key-log for HIGH (¬ІF.5), the anchored map + substrate root for
  TOP (¬ІF.5a, ¬ІF.3). Removing any such coordinate from `вДР` removes it from the ѕГ-algebra that decides the higher
  tier, so the decidable tier can only fall. Stripping is a coarsening, never a refinement.
- **No upward forge (`W1`).** `T(d, вДР)` is never ABOVE the true tier the evidence supports: authority can be
  DENIED but not fabricated (F.3.1 / F.5a вАФ non-membership and anchored time are not producible by the
  publisher). Evidence is necessary, not assertable; there is no header, flag, or self-claim in `рЭТЃ_LIGHT` that
  raises `T`.
- **Consumer floor.** A consumer names a REQUIRED tier `R` and accepts `d` iff `T(d, вДР) вЙ• R`. This is the only
  place the ordering is USED: the floor is a comparison, not a coercion.

**Theorem F.5b (downgrade resistance).** Let a consumer hold floor `R` and let an attacker present `d` with an
eroded `вДР' вКЖ вДР`. Then either `T(d, вДР') вЙ• R` (the surviving evidence still earns `R` вАФ a genuine `R`-grade
document, nothing was gained) or `T(d, вДР') < R`, in which case the consumer REJECTS. There is no third branch:
because acceptance is `T вЙ• R` and never "accept at whatever `T` came out," a strip that drops the tier below the
floor produces a rejection, never a silent accept at the lower tier. *Proof.* Immediate from monotone erosion
(the strip can only lower `T`) and the floor being a total-order comparison against a fixed `R`. Downgrade is the
floor doing its job; it is the consumer's CHOICE of `R`, never a forge succeeding (`W1` blocks the only other
route, raising `T`). вИО

**Corollary F.5b (the two floors are symmetric).** `requireAuthoritative` is the floor `R = HIGH`-authoritative;
`requireAnchored` is the floor `R = TOP`. A verifier MUST implement each as `T(d, вДР) вЙ• R ? accept : reject`. The
rejection NAMES the missing coordinate вАФ a stripped/absent anchor proof вЗТ `E-ANCHOR` (a structural downgrade: the
document cannot reach TOP without a new proof), a non-authoritative identity вЗТ `E-GENESIS`, and an anchor that is
PRESENT and inclusion-valid but whose substrate is unreachable or not-yet-buried вЗТ `INDETERMINATE` (retry, `W1`:
the evidence may still arrive вАФ this is unavailability, not a forgery). Absence of a floor is the LIGHT default,
where `T` is surfaced as-is; the floors exist precisely so a TOP-needing consumer cannot be handed a LIGHT doc.

- **Election вЗТ uniqueness.** The store-NX election (a content-addressed conditional-write: the first writer to
  claim `ust_id` wins, ¬І11 dual-writer) admits AT MOST ONE writer's document to the set that is later anchored.
  So in honest operation `|{ i : ch(dбµҐ) вИИ A_{auth} }| вЙ§ 1`.
- **Exactly one anchored вЗТ that one is canonical.** The others are non-canonical LOSERS вАФ not invalid (each may
  be a perfectly `VALID:HIGH` document; it simply lost the race and was never anchored for this slot). A consumer
  deterministically keeps the anchor-included one.
- **Zero anchored вЗТ `INDETERMINATE` at TOP.** No candidate is in `FвВЬ` yet (the hour is still open, or neither was
  elected). Fork-choice is undecidable at TOP; the consumer resolves at HIGH by other means or WAITS for the hour
  anchor. This is unavailability (`W1`), not a fault.
- **Two or more anchored under ONE authority вЗТ `E-PREV` (equivocation).** `|{ i : ch(dбµҐ) вИИ A_{auth} }| вЙ• 2` with
  distinct `content_hash`es means the operator committed TWO states for one slot into an anchored root under one
  name вАФ a non-repudiable, detectable fault. The anchor makes it PUNISHABLE: the operator SIGNED a root
  containing both, so equivocation is evidence, not deniable ambiguity. (Distinct authorities publishing the same
  `ust_id` are NOT a fork вАФ canonicity is per-authority; each is canonical in its own `A_{auth}`.)

**Proposition F.5c (determinism).** Two consumers with the same candidate set and the same anchor `A_{auth}` emit
the SAME `canonical`. *Proof.* The choice reads only `content_hash(dбµҐ)` (a function of the bytes) and membership
in `A_{auth}` (a function of the shared anchor); neither depends on arrival order or local state. So `canonical`
is a function of `(candidates, A_{auth})` alone. вИО This is exactly the property ¬І11 requires of the dual-writer:
"canonical = anchor-included" turns an operator-side race into a consumer-side FUNCTION вАФ the loser is decided by
the chain, not by whichever document a given agent happened to fetch first.

## F.5d Key-log freshness is the third face of authenticated non-membership

The revocation predicate "key `k` is still valid at time `t`" is `¬ђвИГ e : e revokes k вИІ e вИИ keylog вИІ time(e) в™ѓ t`
вАФ a NON-MEMBERSHIP statement over the key-log, structurally identical to no-fork on the name axis (F.5a) and
snapshot-completeness on the time axis (F.3.1). By Corollary F.3.1 it does NOT follow from any positive view: a
consumer holding a cached key-log `L' вКЖ L` decides only `revoke вИЙ L'` (membership in its own prefix), which is in
`ѕГ(L')`, NOT in the ѕГ-algebra that decides `revoke вИЙ L`. The publisher (or a stale cache) can OMIT the revoking
entry exactly as it can omit a rival genesis. So freshness is earned by bringing the non-membership coordinate
into `вДР`. **An earlier claim вАФ that an anchored key-log HEAD alone settles `¬ђвИГ later entry` and thus earns
`attested` вАФ was UNSOUND (P0-03, external audit): an anchored head proves membership AT its anchor time, not that
it is the LATEST head at the target's time; a revoke that FOLLOWS the anchored prefix is invisible to it (the F.5a
"positive lookup settles ¬ђвИГ" transposition fails here because the key-log has no independent map coordinate, only a
prefix).** Strong key-log freshness is therefore not a single-anchor fact: `corroborated`/`attested` are earned ONLY
through the checkpoint derivation (F.5i/F.5j) вАФ authorization вИІ strict terminality (F.5n) вИІ proven-after ordering вИІ
independent uniqueness. A fetch from the authoritative surface timestamped `вЙ• t` still earns `fresh` (a single-view
report); neither вЗТ `unverified`. Emitting `unverified` (never a forged "valid") and letting the consumer floor on it
(`requireFreshKeylog`) is the F.5b discipline: the strong word is earned by the composed predicate, not bought by
assuming a cached prefix is the whole log.

## F.5e The key-authority process `K_n(t)` вАФ a state machine, not a set (MATH-04, #75)

`W_n` (¬ІF.5, ¬ІF.5a) bundled several facts; one of them вАФ WHICH key is authorized for name `n` at time `t` вАФ is
not a static set but a **process**. Model the key-log as a sequence of events `(e_1, вА¶, e_m)`, each an
`add | rotate | revoke` transition, and define the reducer

  `K_n : (event prefix) вЖ¶ вЯ®active вКЖ Keys, bind вКЖ Keys, revoked : Keys вЗА {retired, compromised}√ЧTimeвЯ©`,

with `active` the keys that may sign the NEXT event, `bind` every key ever authorized (for document binding), and
the transitions: `add(k)` вЗТ `active вИ™ {k}`, `bind вИ™ {k}`; `rotate` (signed by `s`, naming successor `k`) вЗТ
`active вИ™ {k} вИЦ {s}`, `bind вИ™ {k}`, `revoked[s] = (retired, t)`; `revoke(k, r)` вЗТ `active вИЦ {k}`, `revoked[k] =
(r, t)`. The **admissibility invariant** is `signer(e_{i+1}) вИИ active(after e_i)` вАФ an event is well-formed only
if its signer was active in the state the PREVIOUS events produced. This is exactly the missing coordinate: "key
`k` appears somewhere in the log" (`k вИИ bind`) is strictly weaker than "`k` was active when it signed" (`k вИИ
active` at that prefix), and conflating them is the P0-02 class (a revoked / rotated-out key still signing).

**Realization (representation note).** `K_n` is realized by `resolveKeys(genesis, keylog)` вЖТ `{active, validKeys
(=bind), revoked, history, head}`; the invariant is the `active.get(keyId(sig.pub)) === sig.pub` gate per entry;
the closed per-`op` schema is the well-formedness of each event. Document authority reads `bind` (continuity) then
the ¬І12.2 X1 predicate judges `revoked[k]` at the document's time вАФ a query of `K_n` at `t`. **The query time `t`
is the PROVEN anchor upper bound `U` (ROOT 1 / MATH-05, done): `verify` runs in two phases вАФ it verifies the anchor
FIRST, then resolves authority with that proven `U`, so revocation / retirement / freshness are decided against
the chain, not a caller-supplied or absent time.** `K_n(t)` is a WINDOW, `authorized_at(k) вЙ§ U вЙ§ end(k)`: the
lower bound is the new `premature` verdict (a document cannot be proven-anchored before its signing key was
authorized), the upper bound is the X1 retired/compromised predicate. The `authority-at-time` vectors exercise the
window; the `keylog-state` vectors exercise the reducer invariant вАФ both are executable instances a second
implementation runs through its own `K_n` and must match. (The `authorized_at` lower bound uses the key's CLAIMED
authorization time; making it an ANCHORED lower bound is the operator manifest, ROOT 3.)

## F.5f Composite authority is TRANSITIVE вАФ the impersonation fix needs no new signed object (#75 ROOT 3)

An external audit proposed closing the composition holes (an impostor becoming `canonical` / `complete` under a
victim's name, P0-03; off-grid commitment grinding, P0-04) with a new **operator-signed HourManifest**. The math
says that is unnecessary, and shows *why* вАФ resolving two apparent dilemmas ("sign the hour root or not?", "typed
leaf or raw hash?") by direct computation rather than taste.

**Proposition F.5f.1 (impersonation is a per-frame authority failure, not a missing manifest).** A candidate `d`
claiming authority `A` for slot `s` is canonical-under-`A` only if `key(d) вИИ K_A(t)` (¬ІF.5e). The impersonation
verdict is `key(d) вИЙ K_A` вАФ decidable in `ѕГ(K_A)` from the genesis+key-log ALONE (ROOT 1+2), independent of any
anchor or manifest. So `forkChoice`/`verifyStream` must read the RESOLVED authority `A` (the name the resolved key
binds), never the LIGHT `domain_shard` CLAIM in the bytes. The claim is in `рЭТЃ_LIGHT`; the binding is in `ѕГ(K_A)`;
conflating them was the whole bug. No new object closes a ѕГ-algebra gap that `K_A` already decides.

**Proposition F.5f.2 (the hour root carries no information a signature could add).** Let the hour's leaves be the
frames `d_1вА¶d_m`, each already `A`-authenticated (F.5f.1), and let the expected grid be `G = grid(from, to,
cadence)` where `cadence` is `A`-signed (F.5e cadence log). The hour root `R = merkle(d_1вА¶d_m)` and the
completeness predicate `{d_i} в§≥ G` are DETERMINISTIC functions of already-authenticated inputs. A signature over
`R` is therefore in `ѕГ(K_A, {d_i}, cadence)` already вАФ it adds no measurable event (the same reason `content_hash
= H(canon(state))` is not separately signed: it is a function of the signed state). Hence an operator hour root
is correctly UNSIGNED; its trust is (i) the `A`-authenticated leaves, (ii) the `A`-signed cadence, and (iii) the
substrate anchor for TIME + immutability (`FвВЬ`), NOT an operator signature. The prod notary reached this
empirically (git+OTS, unsigned root); F.5f.2 is the reason.

**Corollary F.5f.3 (the typed leaf is redundant).** `content_hash(d) = H("ust:state", canon({ust, state}))` and
`state.id` contains `domain_shard` and `ust_id`; so `content_hash` already COMMITS the coordinate
`(domain_shard, ust_id, value)`. A leaf `H("ust:frame", canon({domain_shard, ust_id, content_hash}))` re-commits
what `content_hash` commits вАФ no separation gained. The raw `content_hash` IS the coordinate-bound leaf.

**What the anchor is still load-bearing for** (not dissolved): (a) **canonicity among `A`'s OWN candidates** вАФ a
dual-writer race yields two `A`-authenticated values for `s`; the one in `A`'s anchored set is canonical
(`forkChoice` via the substrate proof it already takes) вАФ the store-NX election puts exactly one there, вЙ•2 вЗТ
equivocation вЗТ `E-PREV`; (b) **standalone completeness** for a consumer who holds NONE of the frames вАФ the
operator serves the anchored, deterministic (unsigned) hour index; this is an operator-profile artifact (¬І20),
not a new protocol primitive. (c) The remaining `latest-head` freshness (P0-05) is F.5a/F.5d authenticated
non-membership вАФ the one composition problem that genuinely needs the anchored monitorable single-head, tracked
separately. The audit's manifest collapses, by the math, to "per-frame `K_A` + `A`-signed cadence + the existing
substrate anchor" вАФ less machinery, each omission proven, not cut.

## F.5l Checkpoint recovery вАФ a genesis-rooted threshold re-adapts the authority process without breaking well-foundedness (#76 ¬І1.7)

F.5h's authority process halts if the currently-authorized checkpoint key is lost: `Auth(n)` was to be committed by
`CвВЩвВЛвВБ`, and no one can now produce it. F.5l adds a SECOND, genesis-rooted authority source that re-adapts the
process вАФ without self-authorization and without bypassing any other checkpoint predicate.

**A second `рЭУХ_{n-1}`-measurable authority.** Genesis fixes a recovery key set `RK` (role-separated from data and
checkpoint keys, immutable within the epoch) and a threshold `t`. A recovery statement for sequence `n` is authorized
iff `вЙ• t` DISTINCT keys of `RK` sign the BYTE-IDENTICAL typed claim binding `(domain, genesis_epoch, last_accepted =
id(CвВЩвВЛвВБ), effective_sequence = n, replacement_authority)`. The recovered authority `Auth_rec(n)` is then the claim's
`replacement_authority`. Both candidate authorities are measurable in the past: normal `Auth(n)` from `CвВЩвВЛвВБ`, and
`Auth_rec(n)` from `RK вКЖ рЭУХ_0` (genesis) plus a statement bound to `id(CвВЩвВЛвВБ)`. Neither reads `CвВЩ`'s own content, so
`Auth_rec(n) вЙЇ CвВЩ` вАФ the F.4/F.5h adaptedness and well-foundedness hold. Recovery is NOT self-authorization: the LOST
key does not sign; the genesis threshold does. The verifier accepts `CвВЩ` iff its signature matches `Auth(n)` OR
`Auth_rec(n)`, resolved before the signature is trusted (F.5h resolve-before-trust, extended to a two-element set).

**Threshold authorizes ONE replacement but does NOT prove no rival вАФ the conflict must be DETECTED.** An earlier
claim вАФ that two statements naming different replacements are "each below `t`, so neither recovers" вАФ was FALSE
(P0-05, external audit): with OVERLAPPING quorums a single EQUIVOCATING recovery signer makes two conflicting
replacements EACH reach `t` (e.g. `{R1,R2}` for A and `{R2,R3}` for B under 2-of-3, `R2` equivocating). A threshold
tolerates the LOSS of `|RK|вИТt` keys; it does NOT guarantee a unique decision under a Byzantine signer. So the
verifier GROUPS distinct valid signers by the canonical claim they signed and REJECTS вАФ a detectable authority
conflict, never an array-order-dependent pick вАФ if MORE THAN ONE distinct replacement reaches `t`; it also requires
`1 вЙ§ t вЙ§ |RK|` (`t = 0` is not authorization). Threshold is formal AUTHORIZATION of one replacement; the independent
rule that SELECTS it is UNIQUE threshold-attainment, exactly as F.5j/F.5k separate authorization from anti-equivocation.

**Bounded and non-bypassing.** `effective_sequence = n` binds recovery to EXACTLY the next checkpoint; it
re-authorizes the SIGNER coordinate only. `CвВЩ` still passes every other predicate вАФ sequence, `previous_checkpoint`
linkage, rotation exactness, terminality, consistency, external commitment, and (for `attested`) independent
uniqueness. So recovery is a re-rooting WITHIN the F.5h chain, not an escape from it; and because `RK` is
genesis-fixed and role-separated, a normal checkpoint key can never silently replace the dormant emergency roots.

**Realization.** `checkpointRecoveryClaim`/`buildRecoveryStatement`/`verifyCheckpointRecovery` (the typed multisig)
and the recovery branch of `verifyAuthorityCheckpointChain` (`{recoveries, recoveryKeys, recoveryThreshold}` вАФ the
matched signer, normal or recovered, becomes the authority in force for the next sequence).

## F.5m Genesis-epoch transition вАФ a signed re-rooting that extends the adapted process across epochs (#76 audit-8)

F.5h's authority process runs WITHIN one genesis epoch. A new epoch `B` succeeding `A` must re-root the authority
without breaking the causal structure and without a self-declared reset. F.5m is the signed hand-off that does so.

**Adaptedness across the boundary вАФ the destination is a VERIFIED genesis, never a free label (M4.4).** The
transition `ѕД` is a typed statement SIGNED BY epoch A's authority вАФ the authority in force at A's final checkpoint
`F_A` вАФ binding `(domain, from_epoch = A, from_final_checkpoint = id(F_A), to_active_genesis = contentHash(g_B),
to_epoch = H_"ust:genesis-epoch"(to_active_genesis), to_checkpoint_authority, to_initial_sequence)`. The rc.35
round-2 correction: `to_epoch = B` used to be a free string, so a transition could seed an epoch that binds NO
genesis; now `to_active_genesis` is REQUIRED and `to_genesis_epoch` must be CANONICAL to it (the M2 hygiene,
uniform on both sides of the boundary) вАФ with M2 on the checkpoint side, the epoch-initial `CвВАбіЃ` provably LIVES IN
the genesis the transition bound (`active_genesis(CвВАбіЃ) = ѕД.to_active_genesis` is derivable; the explicit chain check
remains as the hash-collision belt). Epoch B's initial checkpoint `CвВАбіЃ` binds `previous_epoch_final_checkpoint =
id(F_A)`, has `sequence = to_initial_sequence`, and is signed by `ѕД.to_checkpoint_authority`. So `Auth(CвВАбіЃ) =
ѕД.to_checkpoint_authority`, and `ѕД` is signed by `Auth(F_A)`, measurable in epoch A's past (`рЭУЧ`-adapted, F.5h) and
bound to `id(F_A)`. Hence `Auth(CвВАбіЃ) вЙЇ CвВАбіЃ`: the adaptedness continues UNBROKEN across the epoch boundary вАФ epoch
A's authority CHOOSES epoch B's, exactly as `CвВЩвВЛвВБ` chooses `CвВЩ`. It is a re-rooting inside the same well-founded
process, not a new independent root.

**No silent reset.** A checkpoint whose `genesis_epoch` differs from the prior WITHOUT a valid `ѕД` is
`INVALID(E-MALFORMED)`: an epoch cannot re-root the authority or reset the sequence on its own say-so. The sequence
reset (to `to_initial_sequence`) is AUTHENTICATED by `ѕД`, never free; `CвВАбіЃ` must bind `id(F_A)` (`E-PREV`) and match
the transition's initial sequence (`E-SEQ`). The domain never changes across `ѕД` (a different domain is a different
publisher, not an epoch transition).

**Composition.** Within epoch B the chain runs normally and may itself invoke recovery (F.5l). Because the map key is
`(domain, genesis_epoch, sequence)` (F.5k), the per-epoch sequence namespace keeps checkpoint uniqueness
well-defined across epochs вАФ a reset to 0 in epoch B does not collide with epoch A's sequence 0.

**Realization.** `epochTransitionClaim`/`buildEpochTransition`/`verifyEpochTransition` (the typed hand-off) and the
epoch-crossing branch of `verifyAuthorityCheckpointChain` (`{epochTransitions}` keyed by `to_genesis_epoch`; the
transition's `to_checkpoint_authority` becomes the resolved signer for `CвВАбіЃ`).

**Conformance (math вЗТ code вЗТ green vector, `packages/ust-protocol/conformance.mjs`).**

## F.5n Strict key-log terminality вАФ head is the LAST entry, not merely a member (#77)

F.5i's `corroborated` needs the checkpoint head to be TERMINAL: the latest key-log entry, with no more-recent
revoking successor. Bare membership `head вИИ root` is NOT terminality вАФ a log that ALSO holds a successor still
contains `head`, so membership is satisfied while the head is stale.

**An earlier construction was UNSOUND (P0-02, external audit).** Committing the key-log as a positioned SMT keyed by
`H(index)` and proving `Inclusion(L-1) вИІ NonMembership(L)` proves only that index `L` is empty вАФ it says NOTHING
about indices `L+1, L+2, вА¶`. Because hashed-index leaves are scattered, `[L, вИЮ)` is not a subtree, so a
single-coordinate non-membership cannot cover the suffix: a prover commits entries at `{0, 2}`, claims `length = 1`,
proves `pos 1` empty, and hides the entry at `pos 2`. Sparse-dictionary absence at one coordinate is not
prefix-contiguity вАФ the claim "non-membership at `L` proves nothing follows" was false.

**Terminality is a SNAPSHOT property of a SIZE-BOUND vector commitment (M4.3).** The key-log is an ORDERED Merkle
over EXACTLY `L` leaves (padded to a power of two with a domain-separated empty leaf), and the committed root binds
the size: `root = H("ust:keylog-commit", {length = L, merkle_root})`. `SnapshotTerminal(root, L, head)` holds iff
WITHIN the fixed committed root of width `next-pow2(L)`: `head` is the leaf at index `L-1`; every RIGHT sibling on
its authentication path (each time the path node is a left child) is the empty-subtree default for its level; the
proof depth is EXACTLY `ceil(log2(width))` with the index fully consumed (the P0-5 refinement вАФ an under-depth proof
recomputes a smaller tree); and the recomputed root equals the committed root. This proves the admissible index
domain of THIS SNAPSHOT is exactly `[0, L-1]` вАФ no committed leaf exists at or beyond `L` *in this root*. It is
deliberately NOT a claim about "the suffix `[L, вИЮ)` forever" (the rc.35 round-2 over-strength): nothing about a
FUTURE, larger snapshot follows from one commitment вАФ that relation is `ChainConsistent` (F.5i/M4.2, append-only
ACROSS snapshots), and currency against real time is `ProvenAfter` (F.5i). Freshness is the COMPOSITION of a
snapshot-terminal, chain-consistent, committed, proven-after, authorized checkpoint вАФ three orthogonal predicates,
none overloaded. Still `SnapshotTerminal вКЛ HeadInRoot`, and вАФ unlike the SMT version вАФ sound against a successor at
ANY index of the committed tree, adjacent or not.

**Realization.** `buildKeylogCommitment(entryHashes)` (ordered size-bound Merkle: `root`/`length`/`head`/`merkle_root`
/`headProof = {index, siblings}` + a `prove(index)`) and `verifyKeylogTerminality({root, length, head}, {headProof})`
(index `= L-1` вИІ every right sibling on the path is that level's empty default вИІ recomputed
`H("ust:keylog-commit", {length, merkle_root})` equals `root`), composed into `deriveCheckpointFreshness` as the
`Terminal` conjunct (F.5i).

**Conformance (math вЗТ code вЗТ green vector, `packages/ust-protocol/conformance.mjs`).**
- terminality holds for honest logs: *"TERM honest length-1 log (head at pos0, nothing at pos1) вЖТ terminal"*, *"TERM honest length-2 log (head at pos1, nothing at pos2) вЖТ terminal"*.
- a hidden/truncated successor is caught: *"TERM strict catches a HIDDEN SUCCESSOR (length lies)"* вАФ now via the P0-5 proof-depth check (siblings вЙ† ceil(log2(width))) as well as the right-subtree-empty test.
- head must sit at the terminal index: *"TERM wrong head at position L-1 вЖТ not terminal"*.

Sound against the P0-02 reproduction (`security-regression.mjs`: a real length-3 log presented as length-1 вЗТ not terminal).

## F.6 Composition вАФ the event algebra

An **anchored existence-and-commitment claim** is an event `A вИИ FвВЬ`; an UNANCHORED signed claim is a document
predicate and enters `FвВЬ` only upon anchoring (a LIGHT document's statements exist before any journal sees
them). Because each `FвВЬ` is a ѕГ-algebra, anchored claims are closed under `вИ©` (AND), `вИ™` (OR), and `бґЬ` (NOT).
The ¬І9 combinators are exactly this closure:

- **Attestation** over constituents `cвВБ,вА¶,cвВЩ` commits to the conjunction `A = AвВБ вИ© вА¶ вИ© AвВЩ` where each `AбµҐ` is
  the EXISTENCE-and-commitment event of record `i` вАФ never its truth, and (per ¬І14a) the attestation alone does
  not even verify the constituents' own signatures; it fixes the SET. The Merkle `root` is the binding of that
  set; walking into the constituents is the referent walk (depth-budgeted, reported).
- **Derivation** has TWO strengths that must not be conflated. `based_on` alone is a **declared** lineage вАФ
  `DeclaredDerivation(d, H)`, the publisher's signed CLAIM that `d` draws on hashes `H`; it does not prove any
  particular function was applied. Only when a verifier holds a deterministic profile `ѕЖ`, the source documents,
  and recomputes вАФ `VerifiedDerivation_ѕЖ(d, H)` вАФ is `d = ѕЖ(H)` measurable, and only THEN does composition of
  measurable maps propagate provenance measurability along the DAG (`E-CYCLE`
  guarantees `ѕЖ` is well-founded / acyclic).

So "combine" never manufactures information вАФ it only takes measurable functions of already-measurable events.

## F.7 The certainty predicate вАФ and where probability is (and isn't) load-bearing

For a committed record with event `A вИИ FвВЬ`, no probability measure is needed to state the certainty claim вАФ
it is a MEASURABILITY statement (formally: under ANY measure, e.g. a point mass on the realized history,
`E[1_A | FвВЬ] = 1_A`; we do not rely on one):

  **1_A is FвВЬ-measurable, and on the event `A` itself `1_A = 1` вАФ i.e. `1_A(ѕЙ*) = 1` for a realized history `ѕЙ*`
  whose inclusion proof checks (not globally on ќ©, where histories without the commitment give `1_A = 0`).**

That is the precise content of the informal `P(record(ѕЙ,t) | FвВЬ) = 1`: *given the filtration at `t`, whether the record was
committed is already known* (measurable) and, for a valid sealed record, certain. **In pure UST every verdict is
deterministic вАФ `0/1` вАФ so the probability is degenerate.** The genuinely load-bearing structure is the
**filtered measurable space `(ќ©, F, {FвВЬ})` plus a deterministic measurability predicate**, not a probability
measure. Rigor demands saying so plainly: UST has no randomness of its own.

Probability becomes non-degenerate at **exactly one boundary** вАФ the measurement gap of ¬ІF.2. Modelling
the deviation `ќµ` (with `M = m(Y, ќµ)`, ¬ІF.2 вАФ no subtraction exists on strings) as a random variable is where a
real (non-`0/1`) measure lives. UST deliberately does **not** model it:
it fixes `M` (with `ќµ` baked in) and stops. So the honest scoping is:

- **filtration вЗТ verification** (deterministic, the heart of UST);
- **probability вЗТ measurement error only** (downstream of UST, in whatever application reasons about how far a
  publisher's measurement `M` might deviate from the truth `Y`).

Conflating the two вАФ reading `P(...) = 1` as if UST asserted something probabilistic about *reality* вАФ is the
formal shape of the "signed = true" over-read the whole protocol exists to forbid.

## F.9 Resource-bounded verification (non-normative; the numbers stay normative in ¬І13)

Information is not the only condition of decidability: a verifier also operates under a finite resource
budget. Alongside its information set `вДР_v`, give every verifier a budget `ѕБ_v = (M_v, T_v, S_v, N_v)` вАФ
memory, time, safe stack depth, and the provenance-node allowance. A verifier lives in the PAIR `(вДР_v, ѕБ_v)`.
For a transcript `R` define the size vector

  `ќЉ(R) = (B, P, D, A, F, W, K)`

вАФ canonical UTF-8 bytes of the signed content, partition count, nesting depth, max array length, direct
reference breadth, per-call walk volume, and the examined key-log epoch length. Let
`C_v : DocSpace √Ч Request вЖТ вДЭвБівВК` be the resource-cost vector of parsing, canonicalising, hashing, signature
verification and the requested walk вАФ components `(memory bytes, time, stack frames, visited nodes)` for
verifier `v`'s implementation вАФ with `ѕБ_v вИИ вДЭвБівВК` and `в™ѓ` the COMPONENTWISE partial order (`a в™ѓ b вЗФ aбµҐ вЙ§ bбµҐ`
for all four); "insufficient resources" means SOME component exceeds its budget. A tier predicate is decidable
by `v` only when BOTH hold: `рЭТЃ_ѕД вКЖ вДР_v` (information) and `C_v(R) в™ѓ ѕБ_v` (computation). Missing information
yields `INDETERMINATE(unavailable)`; insufficient local resources yield `INDETERMINATE(resource_limit)`
(the wire reasons of ¬І15). Neither makes a protocol-valid transcript INVALID.

### F.9.1 Extensive and structural metrics

A metric is a **VOLUME** metric when legitimate publisher data increases it extensively (вЙИ additively under
union), growth leaves the verification CONTROL STRUCTURE unchanged (more of the same work, not different
work), and excess payload admits a commitment-preserving externalization (content-address the bulk, keep the
signed hash). `B` and `P` are VOLUME metrics; per-partition ciphertext bytes are volume too but NOT an
independent coordinate of `ќЉ_V` вАФ they are already included in `B(R)`, and any separate per-partition ciphertext
cap is an admission rule outside `ќЇ` (classification and vector stay aligned, 18th round). A metric is a **STRUCTURE**
metric when increasing it enlarges recursion, branching, traversal or resolution state for EVERY verifier
regardless of publisher authority, and a protocol-native transformation can express the same data without the
increase. `D, A, F, W, K` are STRUCTURE metrics. Hence the law:

  **Authority may enlarge legitimate volume, but cannot make structure cheaper to verify.**

That is why VOLUME bounds are ceremony-declarable and STRUCTURE bounds are protocol-wide laws (¬І13).

### F.9.2 Capacity earned by authority

Let `ќЇвВА = (BвВА, PвВА)` be the universal LIGHT floor, `ќЇ_ABS = (B_ABS, P_ABS)` the protocol ceiling, and let an
authoritative genesis declare `ќЇ_G` subject to `ќЇвВА в™ѓ ќЇ_G в™ѓ ќЇ_ABS` (an out-of-range declaration is not a valid
declaration вАФ the floor applies). The declaration is EFFECTIVE only when its genesis binding lies in the
verifier's authority information: `G_ќЇ вИИ рЭТЬ` and `рЭТЬ вКЖ вДР_v`. Effective capacity:

  `ќЇ(R, вДР_v) = ќЇ_G` if the authoritative genesis is verified and in range; `ќЇвВА` otherwise.

A self-signed declaration does not enlarge capacity because key possession alone never places it in the
name-authority ѕГ-algebra: `рЭТЬ вКД вДТ`. On the wire this is rc.12's TRUSTED GRANT: the grant flows FROM
`resolveAuthority` (or the caller's pin/policy), never from a caller-attached genesis document.

### F.9.3 Bounded-verification verdict

With `S_ABS` the vector of absolute STRUCTURE bounds:

  `V(R; вДР_v, ѕБ_v) =`
  `INVALID(E-BOUNDS)` if `ќЉ_S(R) вЛ† S_ABS` вАФ structure over the absolute law;
  `INVALID(E-BOUNDS)` if `ќЉ_V(R) вЛ† ќЇ_ABS` вАФ volume over the absolute protocol ceiling (checked BEFORE authority:
    a 1 GiB transcript is not-a-UST regardless of any grant вАФ the reference impl already tests `sBytes > ABS`

resources are distinct states.
Transport admission (refusing an over-budget RAW input before decoding) is a fourth, pre-verdict state: a
refusal to start, reported as `resource_limit`, never a statement about the document.

### F.9.4 Portability of the LIGHT floor

Let `рЭУФвВА` be the declared class of baseline environments and `ѕБвВА = inf_{eвИИрЭУФвВА} ѕБ_e`. The floor is PORTABLE
exactly when **every environment meets ITS OWN budget**: `вИА e вИИ рЭУФвВА : sup { C_e(R) : ќЉ_V(R) в™ѓ ќЇвВА, ќЉ_S(R) в™ѓ S_ABS }

information-decidable from its own bytes and computationally decidable by every baseline implementation. This

### F.9.5 Structural escape transformations

An absolute STRUCTURE bound does not reduce expressive power when a protocol-native, commitment-preserving
transformation exists: `4096 = 64¬≤` (a two-level attestation tree carries the full constituent universe at
breadth 64); oversized arrays chunk; deep data restructures; long key histories split into re-genesis epochs;
long walks continue across bounded calls. Each transformation preserves the existence and hash-binding of the
represented state while keeping every single verification inside a universal resource envelope.

### One-paragraph summary

Model the world as a filtered measurable space `(ќ©, F, {FвВЬ})` whose filtration **is** the append-only anchor
journal. Fields are coordinate maps; a record fixes the publisher's *measurement* of them, never the truth. To
**verify** is to test **measurability**: `VALID:LIGHT/HIGH/TOP` names the FINEST of three nested ѕГ-algebras (the

(`рЭТЃ_LIGHT вКЖ рЭТЃ_HIGH вКЖ рЭТЃ_TOP`) with respect to which the record is measurable-and-true вАФ the floor from the
document alone, HIGH adding the name-authority ѕГ-algebra, TOP adding the filtration itself. Two conforming
verifiers agree because the verdict is a single-valued measurable function (I4); a verifier lacking the needed

the measurement-vs-reality gap вАФ which is precisely the line UST refuses to cross.
